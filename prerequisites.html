Prerequisites

Before building your first Kuiper image, ensure your development environment meets these requirements. We‚Äôve designed this guide to help you verify and set up everything needed for a successful build.

Important

Operating System Support

Ubuntu 22.04 LTS is the recommended and officially supported platform. Other Linux distributions may work but could encounter compatibility issues.

Windows is not supported for Kuiper builds.

Required Software

Docker Installation

Docker is essential for creating the isolated build environment. Kuiper requires Docker version 24.0.6 or newer.

Verify if Docker is already installed:

docker --version
# Expected output: Docker version 24.0.6 or higher

If Docker isn‚Äôt installed, follow the official guide:

# Quick installation for Ubuntu
curl -fsSL https://get.docker.com -o get-docker.sh
sudo sh get-docker.sh

For complete installation instructions, visit: https://docs.docker.com/engine/install/

Add your user to the docker group (optional but recommended):

sudo usermod -aG docker $USER
# Log out and back in for changes to take effect

Note

If you skip this step, you‚Äôll need to use sudo with all docker commands, which the build script already handles.

Cross-Architecture Support

To build ARM-based images on x86 systems, you need these packages:

Install cross-architecture support
sudo apt-get update
sudo apt-get install qemu-user-static binfmt-support
What do these packages do?

  • qemu-user-static: Emulates ARM architecture on x86 systems

  • binfmt-support: Allows the kernel to run binaries from different architectures

These enable your x86 computer to build ARM images that will run on Raspberry Pi and other ARM-based devices.

Kernel Module Setup

The binfmt_misc kernel module must be loaded:

Load the required kernel module
sudo modprobe binfmt_misc

Check if the module is loaded:

lsmod | grep binfmt_misc
# Should show: binfmt_misc

To load automatically on boot, add to /etc/modules:

echo 'binfmt_misc' | sudo tee -a /etc/modules

If using Windows Subsystem for Linux:

sudo update-binfmts --enable

Pre-Build Verification

Run this verification script to ensure everything is properly configured:

Complete system verification
#!/bin/bash
echo "üîç Kuiper Prerequisites Verification"
echo "=================================="

# Check OS
if [[ $(lsb_release -rs) == "22.04" ]]; then
    echo "‚úÖ Ubuntu 22.04 LTS detected"
else
    echo "‚ö†Ô∏è  OS: $(lsb_release -d -s) (Ubuntu 22.04 recommended)"
fi

# Check Docker
if command -v docker &> /dev/null; then
    DOCKER_VERSION=$(docker --version | grep -oP '\d+\.\d+\.\d+')
    echo "‚úÖ Docker $DOCKER_VERSION installed"
else
    echo "‚ùå Docker not found - please install Docker"
fi

# Check cross-architecture support
if command -v qemu-arm-static &> /dev/null; then
    echo "‚úÖ qemu-user-static installed"
else
    echo "‚ùå qemu-user-static missing"
fi

# Check binfmt module
if lsmod | grep -q binfmt_misc; then
    echo "‚úÖ binfmt_misc module loaded"
else
    echo "‚ùå binfmt_misc module not loaded"
fi

# Check disk space
AVAILABLE=$(df . | tail -1 | awk '{print $4}')
if [[ $AVAILABLE -gt 10485760 ]]; then  # 10GB in KB
    echo "‚úÖ Sufficient disk space available"
else
    echo "‚ö†Ô∏è  Less than 10GB free space available"
fi

echo ""
echo "If any items show ‚ùå, please address them before building."

Save this script and run it to verify your setup:

chmod +x verify-prereqs.sh
./verify-prereqs.sh

Common Setup Issues

‚ÄúPermission denied‚Äù when running Docker

Problem: Cannot access Docker daemon without sudo

Solutions:

  1. Recommended: Use sudo with the build script (no setup required):

    sudo ./build-docker.sh

  2. Alternative: Add user to docker group:

    sudo usermod -aG docker $USER
# Log out and back in
‚ÄúExec format error‚Äù during build

Problem: Cannot execute ARM binaries

Solution: Ensure cross-architecture support is properly installed:

# Reinstall packages
sudo apt-get install --reinstall qemu-user-static binfmt-support

# Reload module
sudo modprobe -r binfmt_misc
sudo modprobe binfmt_misc

# Verify files exist
ls -la /usr/bin/qemu-arm-static
Build fails with ‚ÄúNo space left on device‚Äù

Problem: Insufficient disk space during build

Solutions:

  • Clean Docker system: docker system prune -a

  • Move to larger partition

  • Free up at least 15GB for comfortable building

Repository path contains spaces

Problem: debootstrap fails with path containing spaces

Solution: Move repository to path without spaces:

# Bad: /home/user/my projects/kuiper-gen
# Good: /home/user/kuiper-gen
mv "/home/user/my projects/kuiper-gen" /home/user/kuiper-gen
